---

Awesome Slide ships two CLIs:

- **`@awesome-slide/cli`** — the scaffolder. One command (`init`) to bootstrap
  a new workspace.
- **`@awesome-slide/core`** — the runtime CLI. `dev`, `build`, `preview`, and
  `sync:skills` for an existing workspace.

After `init`, `package.json` exposes the runtime CLI under standard scripts:

```json
{
  "scripts": {
    "dev": "awesome-slide dev",
    "build": "awesome-slide build",
    "preview": "awesome-slide preview",
    "sync:skills": "awesome-slide sync:skills"
  }
}
```

So in day-to-day use you'll just type `npm run dev`, `npm run build`, etc.
